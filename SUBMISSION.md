## 200-word description (submission text)

**Oniṣòwò** is a self-hostable, open-source AI trading agent that runs in your Telegram, with **Qwen 3.6 Plus** as its brain. **186 skills** — 71 technical indicators, backtest engine, hyperopt, multi-agent debate, recursive self-improvement, adaptive TP/SL, regime detection, and more — wrapped in a 5-command setup.

**Problem.** Existing trading bots are either black-box SaaS (custody your keys, trust their code) or thin LLM wrappers with no trading depth. No open-source agent combines MEV awareness, sybil defense, and a memory that learns from every trade.

**Strategy loop.** Perceive → Decide → Execute → Reflect. Oniṣòwò gathers market + onchain context, asks **Qwen 3.6 Plus** what to do, runs the chosen skills, places the order via Bitget, then writes a memory entry for next time. Every reasoning call — *“is this a good entry?”*, *“what’s the risk here?”*, *“should I cut this position?”* — flows through Qwen.

**Differentiators.**
- **Powered by Qwen 3.6 Plus** — fast, accurate, cost-efficient, with $30 free credits from the Bitget hackathon
- **186 skills** organized in 10 tiers (core trading, risk, technical indicators, market intel, sentiment, strategy, new CEX skills, agent meta, user-facing, utilities)
- **MEV-aware execution** — checks sandwich-attack risk before every swap
- **Sybil counterparty scoring** — refuses to enter positions with 60%+ sybil clusters
- **Recursive self-improvement** — reviews last 7 days of trades, writes a new rule to memory every Sunday
- **Self-hostable** — your keys never leave your machine, no central DB, no custody

**Built with**: Bitget Agent Hub (spot + futures APIs, 58 tools), **Qwen 3.6 Plus** (LLM), Telegram (surface), SQLite (memory), Python 3.10+.

## How to test the demo

1. Open Telegram
2. Search for `@OnisowoBot` (or click https://t.me/OnisowoBot)
3. Send `/start` — you'll see "Ọniṣọwọ́ káàlẹ́" (Yoruba for "welcome, merchant")
4. Try `/status`, `/price BTCUSDT`, `/buy SOL 100`, `/skills`, `/journal`, `/reflect`
5. Type `/llm` to confirm the bot is running on **Qwen 3.6 Plus**

Every command you send is processed by Qwen. Every reasoning line you see in the response was generated by Qwen. The bot is genuinely AI-driven — not a wrapper around hardcoded rules.

## How to self-host (the real differentiator)

```bash
git clone https://github.com/ruzkypazzy/Onisowo.git
cd Onisowo
bash init.sh   # creates venv, installs deps, prompts for .env
nano .env      # fill in 5 env vars (Telegram token, Bitget key, Qwen key)
python main.py # bot is live on YOUR machine, running on Qwen
```

3 minutes. Your keys. Your VPS. Your bot. Running on **Qwen 3.6 Plus**.
